Higher-order functions - Part 1 of Functional Programming in Javascript

  • Higher-Order Functions - functions are values just like strings or numbers function can be assigned to variables or pass into other functions

    • What are they good for? Composition
    • the fact that we can take one function and put it into another function allows us to compose a lot of small functions into bigger functions
    • filter - is a function on the array that accepts another function as its argument which it will use to return a new filter version of the array

    • Filter Example
      var dogs = animals.filter(function(animal) {
      return animal.species === 'dog'})

    • functions that you send into another functions are called Callback function because the host function will call back to them

    • filter will loop through each item in the array and for each item it's going to pass it into the callback function and when it does it will expect the callback function to return either true or false to tell filter whether or not this item should be in the new array

  • You'll write code faster when you're doing functional programming
  • means that you can write software with less bugs and less time
  • filter uses a lot less code then the for loop
  • less code, less logic
  • when we write our software in small simple functions they compose together which allows us to reuse functions all over the place
  • the filter function is just one line of logic
  • the callback function and the filter function merged into eachother

Map - Part 2 of Functional Programming in JavaScript

  • Map is another higher-order function
  • Map will take a callback function.
    • The callback function will pass each item in the animals array.


    • var animals = [
      { name: 'fluff', species: 'rabbit'},
      {name: 'ursula', species: 'dog'}
      ]
    • For-loop example:
      var names = []
      for (var i = 0; i < animals.length; i++) {
      names.push(animals[i].name)
      }
      console.log(names)

    • Map Example:
      var name = animals.map(function(animal){
      return.animal.name })
      console.log(names)

  • Map will include all items in the array but instead it expects the callback function to return a transformed object that it will put into the new array instead of the original animal. In this case the name (as mentioned above)

  • We can use map to create completely new objects

    • Example:
      var name = animals.map(function(animal){
      return.animal.name + ' is a ' + animal.species })
      console.log(names)

  • Short code means almost always less bugs
  • Arrow functions is a new shorter and better syntax for functions in ECMAScript 6.

    • Example:

      var name = animals.map(function(animal) {return.animal.name })

      var name = animals.map((animal) => animal.name)
      • Won't work because its ECMAScript 6 and it's not fully implemented into node and browsers
      • however, if running updated io.js, you could use the harmony arrow function flag (node --harmony_arrow_functions example.js)
